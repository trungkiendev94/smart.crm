SmartCRM AI-Agent
Mô tả: Hệ thống hỗ trợ chăm sóc khách hàng và tự động hóa quy trình kinh doanh (CRM) bằng AI, có khả năng tự đọc tài liệu sản phẩm để trả lời khách hàng (RAG) và tự động kích hoạt các tác vụ hệ thống thông qua AI Agent (Agentic Workflow).
Mô hình kiến trúc của dự án:
🛠️ Lộ trình 4 bước xây dựng và các công nghệ áp dụng
Bước 1: Thiết lập môi trường & Vector Database với PostgreSQL (Học RAG)
Thay vì cài các Vector DB xa lạ, bạn hãy tận dụng ngay PostgreSQL (công nghệ bạn đã biết ) kết hợp với extention pgvector. 
•	Nhiệm vụ:
1.	Cài đặt PostgreSQL có sẵn pgvector (chạy qua Docker cho nhanh ). 
2.	Chuẩn bị một vài file văn bản (Ví dụ: "Chính sách chiết khấu CRM 2026", "Hướng dẫn sử dụng tính năng MeInvoice" ). 
3.	Viết một console app bằng C# để cắt nhỏ văn bản (Chunking), dùng một Open Source Embedding Model (qua HuggingFace hoặc OpenAI API) để biến văn bản thành các dãy số (Vector Embedding) rồi lưu vào PostgreSQL.
•	Từ khóa hiển thị trong CV: PostgreSQL (pgvector), Vector Embeddings, Data Chunking.
Bước 2: Tích hợp Semantic Kernel & Xây dựng RAG Pipeline (Học LLM Integration & RAG)
Semantic Kernel là SDK của Microsoft giúp lập trình viên C# kết nối và điều khiển các Mô hình ngôn ngữ lớn (LLMs) như GPT-4, Claude một cách dễ dàng.
•	Nhiệm vụ:
1.	Tạo một .NET 8 Web API project. Cài đặt thư viện Microsoft.SemanticKernel. 
2.	Kết nối Semantic Kernel với một LLM (bạn có thể mua API key của OpenAI tầm $5 hoặc dùng Ollama để chạy LLM local miễn phí trên máy).
3.	Xây dựng API Chat: Khi người dùng hỏi một câu (Ví dụ: "Hệ thống có hỗ trợ xuất hóa đơn tự động không?"), mã C# của bạn sẽ:
	Biến câu hỏi thành Vector.
	Tìm kiếm trong PostgreSQL các đoạn văn bản có nội dung tương tự (Vector Search).
	Gộp câu hỏi và đoạn văn bản tìm được làm ngữ cảnh (Prompt), gửi cho LLM.
	LLM sẽ trả ra câu trả lời chính xác dựa trên tài liệu bạn cung cấp.

Bước 3: Biến Chatbot thành AI Agent (Học Agentic Workflows)
Đây chính là kỹ năng "đắt giá" mà nhà tuyển dụng Bắc Âu kia đang tìm kiếm. Thay vì chỉ trò chuyện, AI có thể TỰ RA QUYẾT ĐỊNH và gọi các hàm C# của bạn để thực thi hành động.
•	Nhiệm vụ:
1.	Tạo ra các hàm C# (gọi là Kernel Plugins) để thực hiện các tác vụ CRM giả lập như: CreateLead(string name, string email), SendEmailMarketing(string email, string template), CheckInventory(string productId). 
2.	Đánh dấu các hàm này bằng các Attribute của Semantic Kernel (để AI hiểu hàm này dùng làm gì).
3.	Bật tính năng Auto Function Calling của Semantic Kernel.
4.	Thử nghiệm: Khi bạn chat "Có khách hàng tên Kien, email kien@gmail.com muốn quan tâm sản phẩm, hãy tạo Lead và gửi email chào mừng cho họ". AI Agent sẽ tự động phân tích câu nói, tự bóc tách thông tin tên/email, tự động gọi hàm CreateLead rồi tiếp tục gọi hàm SendEmailMarketing. 

Bước 4: Đưa kiến trúc Microservices & Message Queue vào (Showcase thế mạnh cũ)
Để chứng minh bạn là một Senior dày dặn kinh nghiệm , hãy kết nối AI Agent này với hạ tầng Microservices vốn có của bạn. 
•	Nhiệm vụ:
o	Khi AI Agent quyết định gọi hàm CreateLead, thay vì xử lý trực tiếp, nó sẽ đẩy một Message vào Kafka / RabbitMQ. 
o	Một Microservice khác (ví dụ: LeadService) sẽ tiêu thụ (consume) message này để ghi vào DB và xử lý tiếp nhằm đảm bảo hệ thống có hiệu năng cao và không bị nghẽn (scalability).
